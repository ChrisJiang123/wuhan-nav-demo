## 0. 项目一句话

一周内、仅 Android、用 **OpenStreetMap 数据 + MapLibre 渲染 + 自建路由引擎** 做一个武汉城区可演示的驾车导航 Demo。不依赖任何商业地图厂商（高德/百度/天地图均不使用）。

## 1. 锁定的口径（不得擅自更改）

- **地图与路由**：完全开源自搭。禁止引入高德/百度/Google Maps 等商业地图 SDK 或 Web API。
- **平台**：仅 Android。不要写 iOS 特定代码、不要配置 iOS 证书。
- **坐标系**：全链路统一 **WGS-84**。OSM 与 GPS 均为 WGS-84，**禁止引入任何 GCJ-02 / BD-09 坐标转换**。
- **实时路况 / 交通事件**：本周**不做**真实数据源。路况只用道路静态等级着色；事件只支持后台人工录入的演示数据。
- **分发**：内部少数人试用。不写任何公开上架、应用商店审核相关逻辑。

## 2. 技术栈（已定，不要替换）

| 层 | 选型 |
|---|---|
| 移动端 | Flutter（Android only） |
| 地图渲染 | MapLibre GL（flutter 插件） |
| 瓦片 | 自建矢量瓦片（tileserver-gl 或 PMTiles） |
| 路由引擎 | OSRM（默认）或 Valhalla（备选），二选一后不混用 |
| 后端 BFF | Node.js + NestJS（默认） |
| 存储 | PostgreSQL + Redis |
| 运营后台 | 同后端仓库内的轻量 Web 页面 |

如果某个任务需要在 OSRM / Valhalla 之间做选择，默认 **OSRM**（部署更简单、武汉城区驾车够用）。一旦选定，全项目不得混用。

## 4. 绝对禁止（任何任务都不许违反）

1. **禁止硬编码任何密钥 / token / 数据库口令**。一律走环境变量，参考 `.env.example`。
2. **禁止引入商业地图 SDK / API**（见口径）。
3. **禁止引入坐标转换库**（proj4 用于其他用途需在 PR 说明里解释）。
4. **禁止前端直连路由引擎或数据库**，必须经 BFF。
5. **禁止提交真实用户数据、真实坐标日志、`.env` 文件**。
6. **禁止把 OSM 数据文件（.pbf 等大文件）提交进 git**，放 `infra/osm/` 并在脚本里下载。
7. **禁止扩大范围**：不要顺手实现未列在 `docs/TASKS.md` 里的功能，即使你觉得有用。

## 5. 提交前必须满足

- 代码能编译 / lint 通过（前端 `flutter analyze`，后端 `npm run lint`）。
- 触及的逻辑有对应测试或更新了 mock 夹具。
- 不留 `TODO` 阻断主流程；可保留标注清楚的非阻断 TODO。
- PR 描述写清：做了什么、对应 TASKS 里哪个任务编号、如何验证、有无偏离口径。

## 6. 与人类的协作约定

- 你**自主完成单个任务**，但**不自行合并**。完成后提 PR，等人类审阅。
- 遇到口径冲突或需求模糊：**停下来在 PR / 输出里提问**，不要自行假设后继续。
- 涉及路由引擎调优、原生 SDK 联调、真机定位这类“脏活”：尽力产出，但明确标注哪些部分你不确定、需要人工验证。
