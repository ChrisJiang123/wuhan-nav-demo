# PRD — 武汉市内导航 Demo（OSM 开源自搭）

版本 v1.0 · 2026-06-18 · 状态：已冻结

---

## 1. 背景与目标

做一个一周内可交付、仅 Android、完全基于开源栈（OSM + MapLibre + 自建路由）的武汉城区驾车导航演示应用。用于内部演示、录屏、给少数人试用。不替代商业导航，不公开上架。

设计理念延续此前定位：**无广告、极简、克制**。导航是唯一主线，不掺杂任何与出行无关的商业元素。

## 2. 目标用户与场景

- 内部评审 / 管理层 / 少数受邀试用者。
- 典型场景：在武汉城区输入起终点 → 看多条路线 → 进入导航跟随 → 路上遇到情况一键上报。

## 3. 范围

### 3.1 本周做（MVP）

| 编号 | 功能 | 描述 |
|---|---|---|
| F1 | 地图展示 | 武汉城区矢量底图，定位显示当前位置 |
| F2 | POI 搜索 | 搜索起点 / 终点 / 沿途点，基于 OSM 数据 |
| F3 | 路径规划 | 驾车，1–3 条备选，展示时间 / 距离 / 收费估算 |
| F4 | 路线选择 | 对比多条路线差异并选定 |
| F5 | 导航态 | 定位跟随、回中、剩余里程时间、下一动作、基础 TTS、静音 |
| F6 | 路况着色 | 用道路静态等级着色（非实时，须明确标注） |
| F7 | 事件上报 | 一键上报事故/施工/拥堵/封路/违停/坑洼/临检，2 步内完成 |
| F8 | 运营后台 | 演示级查看与审核上报记录 |
| F9 | 弱网兜底 | 最近搜索/路线缓存，断网显示已缓存路线摘要 |

### 3.2 本周不做

实时路况真实数据、交通事件真实数据源、离线地图全量包、离线路由、车道级 AR、室内导航、iOS 端、用户体系/积分/社交、多语言。

## 4. 功能详述

### F1 地图展示
- 启动进入首页即显示武汉城区地图，默认定位到当前位置（无定位权限时回落到武汉市中心）。
- 矢量瓦片来自自建 tileserver，经 BFF `/tiles` 代理或直连。
- 坐标 WGS-84，不做偏转。

### F2 POI 搜索
- 输入关键字，返回 OSM 中匹配的地点（名称、坐标、类别）。
- 接受覆盖落差：演示前需人工补录几个关键武汉 POI 进 fixtures。
- 搜索结果可作为起点/终点/途经点。

### F3 路径规划
- 调 BFF `/route`，BFF 调自建 OSRM。
- 默认驾车，返回 1–3 条路线，每条含：预计时间、距离、收费估算（可粗略/占位）、路线几何。
- 收费为估算或占位，演示中标注“估算”。

### F4 路线选择
- 列表展示各路线差异（时间差、距离差、收费差、特征如“过桥一次/红绿灯多”）。
- 选定后进入导航。

### F5 导航态
- 地图跟随定位、可回中、可切视角、可静音。
- 顶部/底部显示：下一动作（如“800m 后右转进入 XX 路”）、ETA、剩余里程、剩余时间。
- 基础 TTS 中文播报转向。
- 驾驶态一级控件 ≤ 5 个：上报 / 总览 / 结束 / 回中 / 静音。

### F6 路况着色
- 用 OSM highway 等级映射颜色（高速/快速/主干/次干等），**非实时**。
- 代码与 UI 均注明“演示用静态等级，非实时路况”。

### F7 事件上报
- 驾驶态点“上报”→ 选类型（事故/施工/拥堵/封路/违停/坑洼/临检）→ 提交，全程 ≤ 2 步。
- 自动带入当前坐标；描述可语音或文本（可选）；图片可选。
- 提交走 `POST /reports`。
- 支持匿名开关。

### F8 运营后台
- Web 页面列出上报记录，可标记审核状态（待审/通过/驳回）。
- 仅演示级，无复杂权限。

### F9 弱网兜底
- 进入导航前缓存路线摘要、拐点列表、TTS 脚本。
- 断网继续显示已缓存路线，不承诺即时重算。

## 5. 非功能需求

- 性能：首次定位 < 3s，路线计算 < 2s，导航页 ≥ 50 FPS。
- 稳定：路测日 0 个 blocker 崩溃。
- 隐私：最小字段、轨迹默认不留存、匿名上报可选、测试数据定期清理。
- 合规：仅内部小范围；不公开上架；WGS-84 全链路。

## 6. 验收（Demo 通过标准）

能完整走通一次：搜索目的地 → 选路线 → 进入导航跟随一段 → 触发一次上报 → 后台看到该上报。录屏可呈现全流程无阻断。

## 7. 明确的非目标

不追求路径质量达到商业导航水平；不追求 POI 完整；不追求实时性。这些在“已知限制”文档中如实告知。
